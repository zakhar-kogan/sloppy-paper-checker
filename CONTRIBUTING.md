# Contributing

Please open an issue before large changes. Keep source adapters provenance-preserving, treat absence as neutral, and do not add person/venue labels that outrun the evidence. New rubric checks need fixtures representing both concerns and legitimate counterexamples.

Run `make lint`, `make test`, and `make build`. API changes must regenerate `openapi.json` and the extension TypeScript types. Security-sensitive changes should include a regression test.

Contributions are accepted under the repository’s MIT license. Be respectful and avoid uploading non-public manuscripts to issues or test fixtures.
